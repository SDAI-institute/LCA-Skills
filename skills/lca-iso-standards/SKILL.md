Apply ISO 14040/14044 requirements at every phase of the study. This skill is always active — it imposes procedural constraints on how other skills are applied.

Core requirements by phase:

**Goal and scope (ISO 14044 §4.2)**
- State intended application, reasons for carrying out the study, and intended audience.
- Define functional unit with quantified reference flow.
- Document system boundary decisions and cut-off criteria explicitly.
- Identify allocation procedures before modeling begins.

**Life cycle inventory (ISO 14044 §4.3)**
- Track data collection procedures and data quality per input/output.
- Document and justify allocation choices (mass, energy, economic value, system expansion).
- Validate mass and energy balances for unit processes where possible.

**Life cycle impact assessment (ISO 14044 §4.4)**
- Classify elementary flows to impact categories before characterization.
- Note which LCIA phases are mandatory (classification, characterization) versus optional (normalization, grouping, weighting).
- Document which impact method was selected and why.

**Interpretation (ISO 14044 §4.5)**
- Identify significant issues based on contribution analysis results.
- Check completeness, sensitivity, and consistency before drawing conclusions.
- State limitations and conditions under which conclusions are valid.

Agent guidance:
- Flag any study element that would prevent compliance if presented to a third-party reviewer.
- In sensitivity analysis, test the most impactful assumptions identified in interpretation.
- Use ISO 14067 terminology when carbon footprint is the primary outcome.
